Cross-scope findings (deferred to round 2 — behavioral / out-of-owned-files):
- `merge_segments` / `merge_segments_with_class` are byte-for-byte duplicated between `sparrow-engine-cpu/src/detect_audio.rs:748-790` and `sparrow-engine-gpu/src/detect_audio.rs:128-165`. Only 2 sites so does not meet the 3+ dedup rule this round; the natural home is `sparrow_engine_types` (where `AudioRange` already lives) or `sparrow_engine_core`. Flag for round 2 if a third caller appears.
- `sparrow_engine_core::postprocess::try_softmax` (postprocess.rs is outside owned set this round) returns `Classification`, blocking direct reuse from the raw-audio path. A future refactor could split it into a `softmax_probs(&row) -> Vec<f32>` primitive that both `try_softmax` and the new audio top-K post-processor consume.
- `detect_audio.rs` (CPU) line 442: `if outputs.len() == 0` — clippy `len_zero` lint candidate, but `ort::SessionOutputs` may not expose `is_empty()` directly; left untouched pending behavioral verification.
- `resolve_classifier_output` in CPU detect_audio.rs falls back to `outputs[0]` for ANY classifier without a "label"-named output, including multi-head models without that head — the docstring says "single-output classifiers" but the code permits multi-output. Behavioral edge.

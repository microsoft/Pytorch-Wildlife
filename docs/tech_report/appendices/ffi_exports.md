# FFI Export Inventory

All `sparrow_engine_*` C FFI symbols exported by libsparrow_engine with the `ffi` feature enabled. Header file: `sparrow_engine.h` (generated by `cbindgen`). C# binding file: generated by `csbindgen`.

## Phase 1 — core

| Export | Purpose |
|--------|---------|
| `sparrow_engine_engine_new` | Create engine singleton |
| `sparrow_engine_engine_free` | Drop engine (clears `ENGINE_EXISTS`) |
| `sparrow_engine_engine_load_model` | Load model by ID; returns opaque handle |
| `sparrow_engine_engine_unload_model` | Mark model inactive, release session |
| `sparrow_engine_detect` | Single-image detection |
| `sparrow_engine_classify` | Single-image classification |
| `sparrow_engine_pipeline` | Ad-hoc pipeline (detect → crop → classify) |
| `sparrow_engine_detect_free_result` | Free detection result struct |
| `sparrow_engine_classify_free_result` | Free classification result struct |
| `sparrow_engine_pipeline_free_result` | Free pipeline result struct |

## Phase 1 — batch inference

| Export | Purpose |
|--------|---------|
| `sparrow_engine_detect_batch` | Multiple images, progress callback, auto-fallback if model is batch=1 |

## Phase 1 — audio

| Export | Purpose |
|--------|---------|
| `sparrow_engine_detect_audio` | File path + options; returns array of audio segments |
| `sparrow_engine_detect_audio_streaming` | Per-segment callback (for long files / streaming) |
| `sparrow_engine_audio_result_free` | Free audio result struct |

## Phase 3 — standalone (no Engine)

6 standalone FFI exports added in Phase 3. These do not require an `Engine` — they operate on file paths or byte buffers directly.

| Export | Purpose |
|--------|---------|
| `sparrow_engine_hash_file` | SHA-256 file hash |
| `sparrow_engine_day_night` | Day/night classification from image path |
| `sparrow_engine_image_brightness` | BT.709 luma brightness of image |
| `sparrow_engine_verify_model` | Verify model checksum against manifest |
| `sparrow_engine_hash_result_free` | Free hash result struct |
| `sparrow_engine_verify_result_free` | Free verify result struct |

## Phase 3 — engine wrappers

3 thin wrappers that delegate to standalone functions but carry an `Engine` reference for API consistency with other engine-scoped operations.

| Export | Purpose |
|--------|---------|
| `sparrow_engine_engine_verify_model` | Engine-scoped wrapper around `sparrow_engine_verify_model` |
| `sparrow_engine_engine_model_info` | Extended `ModelInfo` with Phase 3 fields |
| `sparrow_engine_engine_list_models_extended` | List models with extended info |

## Symbol leak check

Phase 1 handoff verified: 23 `sparrow_engine_*` symbols exported from `cdylib`, 0 ORT symbols leaked. Source: `docs/master_plan.md § Phase 1`.

Run-time check (Linux):

```bash
nm -D libsparrow_engine.so | grep ' T ' | grep -c '^.* T sparrow_engine_'
# expect: 32 (23 Phase 1/2 + 9 Phase 3)
nm -D libsparrow_engine.so | grep ' T ' | grep -c '^.* T Ort'
# expect: 0
```

## Freshness anchor (2026-05-13)

**Phase 3.8 Phase C Wave 4b lock (2026-05-06)**: 32-symbol `sparrow_engine_*` exported set is invariant across CPU and GPU flavors; G5 acceptance gate enforces. Both `sparrow-engine-cpu` and `sparrow-engine-gpu` ship cdylibs as `libsparrow_engine.so` with byte-identical `sparrow_engine.h` headers and identical `SparrowEnginePixelFormat` ABI symbol set (verified per `docs/changelog.md` audit-fix-2 entry). The Phase-1 + Phase-3 export tables above remain accurate as the constituent breakdown of the 32-symbol surface; future export additions/removals must preserve cross-flavor parity. See `CLAUDE.md`, `docs/master_plan.md § Phase 3.8`, and `docs/review/phase3.8-phase-c/round_01/acceptance_gates.md`.

# Appendices

Reference material for the main tech-report sections. Structured so section authors can link into specific files rather than duplicating content.

## Files

| File | Content |
|------|---------|
| `ffi_exports.md` | Full list of `sparrow_engine_*` C FFI exports, grouped by category |
| `cli_commands.md` | `spe` CLI command inventory with flag matrix |
| `http_endpoints.md` | sparrow-engine-server HTTP endpoint list (15 endpoint/method surface) |
| `open_questions.md` | Tagged `QUESTION:` items from the main sections awaiting resolution |
| `benchmark_logs/` | Raw benchmark output (pointers to `docs/benchmarks.md` for now) |

## Conventions

- Appendices are **lookup tables**, not prose. Use tables.
- Content belongs in the main section first. An appendix exists when the table is too long for the main section to carry inline.
- Links from main sections use relative paths: `appendices/cli_commands.md`.

**Decisions locked in during design.**

- v3: NMS in ONNX graph, not in libsparrow_engine. Model validation at load time rejects non-conforming models.
- v3: Inference is write-only to the database; never reads. `halt_on_store_failure` flag planned for Phase 4.
- v3: Idempotent writes via `UNIQUE(media_hash, model_id)` — Phase 4.
- v3: Normalized bbox `[0, 1]` at every public API boundary.
- v4: Engine is a process singleton; second `Engine::new()` returns `EngineAlreadyExists`.
- v4: Opaque model handles at FFI boundary with `Weak` + `AtomicBool` + `Arc` safety pattern.
- v4: libsparrow_engine owns all preprocessing and postprocessing (eliminates consumer divergence).
- v4: 2 core crates (`libsparrow_engine` + `sparrow-engine-server`) with mandatory `ffi` feature flag; Phase 2.5 added 2 consumer crates (`sparrow-engine-cli` + `sparrow-engine-python`) for a workspace total of 4.
- v4: No reserved fields in FFI structs. Function versioning via `_v2` suffix for ABI evolution.
- v4: `cbindgen` + `csbindgen` generate C and C# headers, feature-flag-gated in `build.rs`.
- v4: Golden reference outputs generated by libsparrow_engine itself, not by Python / PIL.

## Phase 2.5 — spe CLI + Python package (MVP)

**Scope.** CLI binary and Python package with a consistent 8-function API. `sparrow-engine-python` uses PyO3 direct bindings to libsparrow_engine (no HTTP server needed). `sparrow-engine-cli` produces the `spe` binary with static ORT, ~35 MB, distributed via GitHub Releases.

**Status.** Complete.

**8 MVP functions (identical in CLI and Python).**

| Category | Functions |
|----------|-----------|
| Inference (4) | `detect`, `classify`, `detect_audio`, `pipeline` — each accepts single file, list, or directory; always returns `list[Result]` |
| Informational (3) | `list_models`, `model_info`, `active_device` |
| Configuration (1) | `init` (optional explicit; auto on first call with `Device::Auto`) |

**Key design.**

- **Auto-load models.** No separate `load_model()` call. Models loaded lazily on first inference. Engine caches sessions.
- **Ad-hoc pipeline.** `pipeline(img, detector="mdv6", classifier="speciesnet")` — no pre-defined TOML required. Detect → crop → classify in one call.
- **GPU default.** `Device::Auto` probes CUDA first, falls back to CPU. `SPARROW_ENGINE_DEVICE=auto` is the default.
- **`sparrow-engine-client` vs `sparrow-engine-python`.** `sparrow-engine-client` is the HTTP SDK for Sparrow Web. `sparrow-engine-python` is the PyO3 direct-bindings package. Distinct packages; no overlap.
- **`sparrow-engine-python` CPU wheel** uses `load-dynamic` + depends on the pip `onnxruntime` package. Avoids ORT dual-loading conflict.
- **fork() safety.** `ENGINE_EXISTS` AtomicBool leaks to forked children. Python `multiprocessing` must use `spawn`, not `fork`. Documented limitation.

**Audit-fix.** 6 critical/serious fixes applied during Phase 2.5 audit: `cmd_init` global flags, `_get_engine()` thread safety, per-file error continuation, exit 1 on empty input, error counting, CSV headers. Report path: `docs/review/historical/audit-fix-historical/audit-fix-r1/`.

**Phase 2.5 Engine Benchmark (direct inference, no HTTP, RTX 6000 Ada, 100 images).**

| Engine | Device | Per-image (mean) | Per-image (median) | Detections |
|--------|--------|------------------|--------------------|------------|
| libsparrow_engine (Rust) | GPU | 71.5 ms | 63.0 ms | 217 |
| Python ORT | GPU | 89.2 ms | — | 218 |
| Python ORT | CPU | 4569.8 ms | 4564.8 ms | 218 |
| libsparrow_engine (Rust) | CPU | 4939.8 ms | 4940.5 ms | 217 |

Rust GPU is 1.25× faster than Python ORT on the non-inference portion (image decode + preprocessing + postprocessing). CPU: same ORT C library dominates (~99% of wall time); Rust is ~7.5% slower on the CPU decode path. 1-detection difference is letterbox rounding in preprocessing.

## Phase 3 — Utilities + local model catalog

**Scope.** Utility functions (hashing, day/night classification, unified visualization, batch statistics, export) + local model catalog (integrity verification, extended model info, manifest schema extension). ~895 LOC in design, 6 new modules, 9 new FFI exports, 1 new dependency (`sha2`).

**Status.** Complete. Final audit-fix CONVERGED at Round 5, 2026-04-20.

**Design.** 3 rounds, 4 architects + inquisitor, all decisions 4:0 unanimous. Final spec: `docs/design/phase3/final_design.md`.

**New modules in libsparrow_engine.**

| Module | Purpose |
|--------|---------|
| `hash.rs` | SHA-256 file hashing (64 KB chunk) |
| `daynight.rs` | BT.709 luma brightness heuristic; threshold > 85 on [0, 255] |
| `viz.rs` | Unified visualization: detection bboxes, classification labels, pipeline (detect + classify), overhead dots (HerdNet/OWL-T), audio segments, audio confidence heatmap |
| `stats.rs` | Batch detection statistics (per-category counts, min/max/mean confidence) |
| `export.rs` | MegaDet v1.5 JSON + COCO JSON + CSV |
| `catalog.rs` | Model integrity verification (size + SHA-256), extended model info |

**Manifest schema extension.** `onnx_sha256`, `onnx_size_bytes`, `version`, `description` added to `[model]`. `#[serde(default)]` for backward compatibility.

**9 new FFI exports.** 6 standalone (no Engine needed): `sparrow_engine_hash_file`, `sparrow_engine_day_night`, `sparrow_engine_image_brightness`, `sparrow_engine_verify_model`, `sparrow_engine_hash_result_free`, `sparrow_engine_verify_result_free`. 3 engine wrappers: `sparrow_engine_engine_verify_model`, `sparrow_engine_engine_model_info`, `sparrow_engine_engine_list_models_extended`.

**Consumer wiring.**

- CLI: 11 commands (`detect`, `classify`, `detect-audio`, `pipeline`, `models list`, `models info`, `models verify`, `device`, `init`, `hash`, `day-night`). Flags on inference commands: `--visualize`, `--output-dir`, `--format`, `--summary`, `--export-format`, `--export-output`, `--recursive`.
- Python: 14 public functions (8 MVP + 6 Phase 3 standalone). Phase 3 functions `hash_file`, `day_night`, `verify_model`, `summarize`, `visualize`, `export` do not require an Engine (route to `#[pyfunction]` in `_sparrow_engine_core`, bypass `_get_engine()`).

**Testing.** 173 libsparrow_engine tests at `dev@09ee0aa` (Phase-3-baseline 172 + 3 S7 per-category `confidence_min`/`max` tests in Phase 3.5 W1 S7 grew the suite to 175; -2 dropped during later consolidation; net 173 — see `08_validation_and_testing.md` L11), 45 sparrow-engine-cli tests (was 38), 24 sparrow-engine-python tests (was 20), 21 sparrow-engine-client tests (added by Phase 3 final audit-fix). 69/69 manual tests pass on CPU (2026-04-18) and GPU (2026-04-20). 17 MT issues + 5 decisions logged; 8 initially carried to Phase 3.5 backlog (later grew to 12; see `11_phase_3.5_planned.md`).
